# Agent Integration

The fastest way to add ScreenCI is to let a coding agent do it. ScreenCI
publishes an agent-readable integration brief at
[`/integrate.md`](https://screenci.com/integrate.md). Point your agent at it and
describe the flow you want, and the agent scaffolds ScreenCI, authors a video,
and records it.

## Start with one paste

Copy the ready-made prompt (with your one-time setup token) from the Get Started
page in the ScreenCI app, or paste this into Claude Code, Codex, Cursor, or any
other coding agent:

```text
Integrate ScreenCI by fetching https://screenci.com/integrate.md and following its steps (use the one-time setup token otp_your_token when it runs init), then add a screenci video about <feature>
```

Replace `<feature>` with whatever you want the video to show off. Add `@files`
or a URL so the agent has context to work from, for example
`add a video about the onboarding flow in @src/routes/onboarding`.

## What the agent does

The brief walks the agent through these steps:

1. **Scaffold and connect the project.** It runs
   `npm init screenci@latest <setup-token> -- --yes` at the repository root. The
   one-time setup token is exchanged for your organization's `SCREENCI_SECRET`,
   which init writes into `screenci/.env`, so the project is connected on the
   free tier with no browser sign-in. The `--yes` flag runs the command
   non-interactively (defaulting the project name to the repository root
   directory name). This installs the `screenci` package, scaffolds
   `screenci.config.ts` and `videos/`, and installs the `screenci` agent skill
   at the repository root. Without a token, the agent instead tells you to copy
   `SCREENCI_SECRET` from your secrets page into `screenci/.env`.
2. **Author a video** for the flow you described, including narration via
   `video.narration({...})`, with setup wrapped in `hide()` and the demo
   driven with visible clicks.
3. **Verify, then record.** It runs `npx screenci test` until the script is
   green, then `npx screenci record` to capture the final video. Free renders
   include a ScreenCI watermark; upgrading removes it.

## How the agent gathers context

To write an accurate script, the agent works from whatever context you give it:

- **Referenced files.** Point it at routes and components with `@files` so it
  learns the real selectors and flow from your source.
- **Live URLs.** Give it a URL when the app is deployed and you do not want to
  hand it the source.
- **The `playwright-cli` skill.** ScreenCI can install an optional
  `playwright-cli` skill that lets the agent open and explore a live site before
  authoring the script. This is the best path when you only have a URL.

If you would rather generate a first draft yourself before involving an agent,
see [Playwright codegen](/docs/video-script-basics#generate-a-first-draft-with-codegen).

## Connecting the project

The one-time setup token connects the project without a browser sign-in. Copy it
(and the ready-to-run command) from the Get Started page, then run
`npm init screenci@latest <setup-token>`. Init exchanges it for your
`SCREENCI_SECRET` and writes it into `screenci/.env`, so `record` uploads
immediately. The token is single-use and short-lived.

No token to hand? Copy `SCREENCI_SECRET` from your secrets page into
`screenci/.env` (this is also the value you set for CI runs). If the secret is
missing, `record` prints guidance and exits without recording.

The upload contains the raw recording, not your source code. See
[Overview](/docs) for how recording and rendering are split.
