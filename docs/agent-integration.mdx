# Agent Integration

The fastest way to add ScreenCI is to let a coding agent do it. ScreenCI
publishes an agent-readable integration brief at
[`/integrate.md`](https://screenci.com/integrate.md). Point your agent at it and
describe the flow you want, and the agent scaffolds ScreenCI, authors a video,
and records it.

## Start with one paste

Paste this into Claude Code, Codex, Cursor, or any other coding agent:

```text
Integrate ScreenCI by fetching https://screenci.com/integrate.md and following its steps, then add a screenci video about <feature>
```

Replace `<feature>` with whatever you want the video to show off. Add `@files`
or a URL so the agent has context to work from, for example
`add a video about the onboarding flow in @src/routes/onboarding`.

## What the agent does

The brief walks the agent through these steps:

1. **Scaffold the project.** It runs `npm init screenci@latest -- --yes` at the
   repository root. The `--yes` flag is required so the command runs
   non-interactively instead of blocking on prompts, defaulting the project name
   to the repository root directory name. This installs the `screenci` package,
   scaffolds `screenci.config.ts` and `videos/`, and installs the `screenci`
   agent skill at the repository root.
2. **Kick off sign-in in parallel.** Right after init, it runs
   `npx screenci login`, which opens the sign-in link in your browser (and
   prints it as a fallback). This does not block: the agent keeps working while
   you sign in. Only the final recording needs sign-in, so getting it in front
   of you early means `record` will not have to wait.
3. **Author a video** for the flow you described, including narration via
   `video.narration({...})`, with setup wrapped in `hide()` and the demo
   driven with visible clicks.
4. **Verify, then record.** It runs `npx screenci test` until the script is
   green, then `npx screenci record` to capture the final video. If you have not
   finished signing in yet, `record` waits for it and continues automatically
   once you do.

## How the agent gathers context

To write an accurate script, the agent works from whatever context you give it:

- **Referenced files.** Point it at routes and components with `@files` so it
  learns the real selectors and flow from your source.
- **Live URLs.** Give it a URL when the app is deployed and you do not want to
  hand it the source.
- **The `playwright-cli` skill.** ScreenCI can install an optional
  `playwright-cli` skill that lets the agent open and explore a live site before
  authoring the script. This is the best path when you only have a URL.

If you would rather generate a first draft yourself before involving an agent,
see [Playwright codegen](/docs/video-script-basics#generate-a-first-draft-with-codegen).

## Signing in

Only the final recording needs sign-in, not authoring or testing. Run
`npx screenci login` right after init to open the sign-in link in your browser
(it prints the link too, in case no browser can open). The command returns
immediately, so the agent keeps authoring and testing while you sign in. The
link stays valid for 24 hours, and a later `record` reuses it.

When sign-in is still pending, `npx screenci record` prints the same link, waits
for it, and records automatically once you finish, so the agent surfaces the link
to whoever can sign in and then keeps the command running. Waiting is the default
off-CI; if it times out before sign-in, `record` exits non-zero (the link stays
valid, so rerun once signed in). Under CI (`CI=true`) it does not wait, so set
`SCREENCI_SECRET` ahead of time for CI runs (this also skips sign-in entirely
anywhere).

The upload contains the raw recording, not your source code. See
[Overview](/docs) for how recording and rendering are split.
